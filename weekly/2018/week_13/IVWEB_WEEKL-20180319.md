<h1 id="#title_22" >23、开发者噩梦：欧盟希望过滤上传到互联网的所有代码</h1>

[https://www.oschina.net/news/94316/eu-copyright-proposal-could-limit-github-code](https://www.oschina.net/news/94316/eu-copyright-proposal-could-limit-github-code)
<p><img src="https://static.oschina.net/uploads/space/2018/0318/081842_5gxc_2720166.png"/></p><p>欧盟的一项提案要求托管内容的所有平台检查所有上传的内容有无侵犯版权，这可能会使软件的成本更高昂。<br/></p><ul class=" list-paddingleft-2"><li><p>欧盟考虑的一项版权提案要求托管由用户上传的大量内容的任何平台扫描所有内容，以确保没有侵犯版权。</p></li><li><p>该提案旨在防止媒体盗版，但可能会给使用代码仓库GitHub等服务的广大开发者带来严重影响，按照新法律GitHub将被迫过滤代码。</p></li></ul><p>欧盟考虑的一项版权提案旨在让盗版者更难共享内容和媒体，但是这张网撒得太广了，可能无意中阻碍程序员。<br/></p><p>正如GitHub指出的那样，自动过滤代码对于独立程序员和大企业程序员来说都是毁灭性的。面临的问题可能包括：误报/漏报、丢失依赖项、许可证混淆以及不必要的负担阻碍创新。<br/></p><p>版权提案仍在讨论中，已经有人开始竭力阻止实施。<br/></p><p><strong>提案有什么样的内容？为何让码农们深为担忧？</strong><br/></p><p>该提案的第13条专门涉及自动内容过滤器的实施，这也是让GitHub及其他欧洲程序员们深为担忧的部分。<br/></p><p>第13条规定：“如果提供商存储用户上传的大量作品或其他主题，并允许公众访问这些内容”，须与版权所有者合作，实施措施以防止内容被非法共享。<br/></p><p>&nbsp;“那些措施（比如使用有效的内容识别技术）应该是适当的、相称的，”该提案表示。第13条进一步提到了内容识别技术是发现版权侵犯行为的最佳实践的一部分，因此过滤器很可能是通过的任何新版权法的最终条文的一部分。<br/></p><p>然而，哪些类型的内容要过滤却没有明确规定。撒一张这么大的网让GitHub和Save Code Share的成员们深为担忧，自由软件基金会欧洲分会和开放论坛欧洲共同发起了Save Code Share，这场请愿活动旨在阻止第13条实施，呼吁欧盟的政策制定者重新考虑或摈弃第13条。他们认为，对于处理代码、文档、音频和视频的开发者来说，版权合规检查完全实现自动化是不可行的。<br/></p><p>如果真是那样子，代码共享平台就需要招聘众多的人员帮助做好版权合规工作，Facebook和YouTube等社交媒体平台之前就是这么做的。<br/></p><p>不管怎样，通过使用Git及其他软件工具而共享的大量内容需要GitHub等面向开发的平台公司实施自动化过滤机制，确定什么内容可以共享、什么不可以共享。<br/></p><p>GitHub的政策经理艾阿•沃尔默（Abby Vollmer）在发给IT外媒The Register的电子邮件中表示，如果美国开发者/开发商在欧洲开展工作/业务，需要遵守同样的这些规定。<br/></p><p>GitHub认为，对于软件开发者而言，由于应用程序常常牵涉许多不同的贡献者和不同层的代码（这些代码也许采用不同的许可证），可能出现误报或漏报这个问题显得尤为突出。<br/></p><p>此外，要求代码共享平台删除没有许可证的代码<br/></p><p><strong>“审查机器”是否来自欧盟？</strong><br/></p><p>欧洲议会议员朱莉娅•里达（Julia Reda）细述了版权提案中另外一些令人担忧的条文，它们适用于编程平台及其他内容共享网站，这些平台和网站都将受到该提案的影响：<br/></p><ul class=" list-paddingleft-2"><li><p>平台需要为它们托管的所有版权保护的内容获得许可证，她认为这几乎是不可能的，对于代码共享平台而言尤为如此。</p></li><li><p>平台需要阻止版权保护的内容被上传，她认为这就要求使用内容过滤器。</p></li><li><p>因为上传/阻止的内容将被去除标识符，内容过滤器无法处理任何个人数据，她表示因而不可能对针对过度过滤的纠错进行归档。</p></li><li><p>扫描的文件将针对所有上传的内容予以检查，里达认为此举违反了欧盟在一般监视方面的禁令。</p></li><li><p>过滤不适用于在线市场（如eBay）、互联网服务提供商（ISP）、研究资料库或云存储服务。里达指出，这些服务并非被免除许可证要求，因令人混淆的语言而容易面临诉讼。</p></li></ul><p>围绕版权提案的争论从未停过，所以现在判断它的一些更广义、更苛刻或不一致的部分条文是否会成为法律还为时尚早。<br/></p><p>这项合理的提案其初衷是确保表演者、创作者和出版者获得作品的报酬，但未必很适合其他社区；谁想发表意见，欢迎联络欧洲议会议员、理事会成员和欧盟委员，确保其心声被听到。</p><p>来自：</p>
