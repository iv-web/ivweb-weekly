曹轩
[http://www.infoq.com/cn/presentations/intelligent-fault-location-system-based-on-log-trace?utm_campaign=infoq_content&utm_source=infoq&utm_medium=feed&utm_term=global](http://www.infoq.com/cn/presentations/intelligent-fault-location-system-based-on-log-trace?utm_campaign=infoq_content&utm_source=infoq&utm_medium=feed&utm_term=global)
<img src="https://res.infoq.com/presentations/intelligent-fault-location-system-based-on-log-trace/zh/mediumimage/caoxuan270-1517055815339.jpg"/><p>传统的trace系统面对大规模系统故障时有以下几个问题：
1.日志回收处理能力强，但对于单PV的精细分析定位受限于人工分析速率，通常在分钟级每PV；
2.大规模复杂异常时，抽样个别PV的定位结论，容易以偏概全，缺少汇聚回归；
3.依赖于人的经验；

1.基于GBDT的单PV根因预测模型；
2.数据多维度汇聚与维度间信息熵排序；
3.智能抽样系统；
4.数据获取异步化、维度逐级下钻等一系列工程优化；
最终能够实现WQPS/sec的PV根因定位能力，并能够根据根因做统计上的多维度汇聚，该系统应用于百度核心搜索系统，极大的提升了重大异常问题定位效率。
</p> <i>By 曹轩</i>

<h1 id="#title_25" >26、活动报告平台 Kibble 跳过孵化直接成为 Apache 顶级项目</h1>

[https://www.oschina.net/news/93042/kibble-to-be-apache-tlp](https://www.oschina.net/news/93042/kibble-to-be-apache-tlp)
<p>Apache 软件基金会宣布 Apache Kibbl 成为 Apache 顶级项目&nbsp;(TLP)。</p><p>Apache Kibble 是一个活动报告平台，用于收集、汇总、分析和可视化软件项目和社区中的活动。借助 Kibble ，用户可以通过在特定时间段内映射的详细视图，来跟踪项目的代码、讨论和问题等。</p><p>大多数的 Apache 项目在成为顶级项目之前，都需要经过孵化器孵化，满足一系列质量要求之后才可毕业。从孵化器里毕业的项目，要么独立成为顶级项目，要么成为其他顶级项目的子项目。不过，Apache Kibble 是直接作为顶级项目进入 Apache 软件基金会的。作为其进入资格，Apache Kibble 必须直接满足  的许多要求，包括项目的代码、版权、许可证、发行版本、利益观、独立性等等。</p><p><img height="409" data-cke-saved-src="https://static.oschina.net/uploads/space/2018/0202/150603_tCR3_2896879.png" src="https://static.oschina.net/uploads/space/2018/0202/150603_tCR3_2896879.png" width="600" border="0" hspace="0" vspace="0" title="" style="width: 600px; height: 409px;"/></p><p>Apache Kibble 也可以看成是一个 Snoot 的开源版本，目前已被大量 Apache 项目采用，并被 Apache 软件基金会用于其官方报告中，比如 ASF 年度报告。</p><p>Apache Kibble 的副总裁 Rich Bowen 表示：“我们对解决困难问题充满激情，特别是当它们会涉及到定义和衡量一个项目的成功时。正因为这样很难，所以我们希望能提供一套工具，能够定义成功并跟踪取得成功的进展，且以最适合其社区的方式进行。Apache Kibble 由此应运而生。”</p>
