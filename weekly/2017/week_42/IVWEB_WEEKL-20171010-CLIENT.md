<h1 id="#title_30" >31、你懂 Linux 吗？关于 Linux 你可能不是非常了解的七件事</h1>

[https://www.oschina.net/news/89404/top-7-things-linux-may-not-known-far](https://www.oschina.net/news/89404/top-7-things-linux-may-not-known-far)
<p><img src="https://static.oschina.net/uploads/space/2017/1009/080930_bC9G_2720166.jpg" width="450" height="338" border="0" hspace="0" vspace="0" title="" style="width: 450px; height: 338px;"/></p><p>使用 Linux 最酷的事情之一就是随着时间的推移，你可以不断获得新的知识。每天，你都可能会遇到一个新的实用工具，或者只是一个不太熟悉的奇技淫巧，但是却非常有用。这些零碎的东西并不总是能够改变生活，但是却是专业知识的基础。</p><p>即使是专家，也不可能事事皆知。无论你有多少经验，可能总会有更多的东西需要你去学习。所以，在这儿我列出了七件关于 Linux 你可能不知道的事情。</p><h3>一个查找命令历史的交互模式</h3><p>你可能对&nbsp;<code>history</code>&nbsp;命令非常熟悉，它会读取 bash 历史，然后以编号列表的方式输出到标准输出（<code>stdout</code>）。然而，如果你在&nbsp;<code>curl</code>&nbsp;命令的海洋里寻找一个特定的链接（URL），那么这个列表并不总是那么容易阅读的。</p><p>你还可以有另一个选择，Linux 有一个交互式的反向搜索可以帮助你解决这个问题。你可以通过快捷键&nbsp;<code>ctrl+r</code>启动交互模式，然后进入一个交互提示中，它将会根据你提供的字符串来向后搜索 bash 历史，你可以通过再次按下&nbsp;<code>ctrl+r</code>&nbsp;向后搜索更老的命令，或者按下&nbsp;<code>ctrl+s</code>&nbsp;向前搜索。</p><p>注意，<code>ctrl+s</code>&nbsp;有时会与 XON/XOFF 流控制冲突，即 XON/XOFF 流控制也会使用该快捷键。你可以通过运行&nbsp;<code>stty -ixon</code>&nbsp;命令来禁用该快捷键。在你的个人电脑上，这通常是有用的，但是在禁用前，确保你不需要 XON/XOFF 。</p><h3>Cron 不是安排任务的唯一方式</h3><p>Cron 任务对于任何水平的系统管理员，无论是毫无经验的初学者，还是经验丰富的专家来说，都是非常有用的。但是，如果你需要安排一个一次性的任务，那么&nbsp;<code>at</code>&nbsp;命令为你提供了一个快捷的方式来创建任务，从而你不需要接触 crontab 。</p><p><code>at</code>&nbsp;命令的运行方式是在后面紧跟着你想要运行任务的运行时间。时间是灵活的，因为它支持许多时间格式。包括下面这些例子：<br/></p><pre class="brush:shell;toolbar: true; auto-links: false;">at&nbsp;12:00&nbsp;PM&nbsp;September&nbsp;30&nbsp;2017
at&nbsp;now&nbsp;+&nbsp;1&nbsp;hour
at&nbsp;9:00&nbsp;AM&nbsp;tomorrow</pre><p>当你以带参数的方式输入&nbsp;<code>at</code>&nbsp;命令以后，将会提示你该命令将在你的 Linux 系统上运行。这可能是一个备份脚本，一套维护任务，或者甚至是一个普通的 bash 命令。如果要结束任务，可以按&nbsp;<code>ctrl+d</code>&nbsp;。</p><p>另外，你可以使用&nbsp;<code>atq</code>&nbsp;命令查看当前用户的所有任务，或者使用&nbsp;<code>sudo atq</code>&nbsp;查看所有用户的任务。它将会展示出所有排定好的任务，并且每个任务都伴有一个 ID 。如果你想取消一个排定好的任务，可以使用&nbsp;<code>atrm</code>&nbsp;命令，并且以任务 ID 作为参数。</p><h3>你可以按照功能搜索命令，而不仅仅是通过名字</h3><p>记住命令的名字非常困难，特别是对于初学者来说。幸运的是，Linux 附带了一个通过名字和描述来搜索 man 页面的工具。</p><p>下次，如果你没有记住你想要使用的工具的名称，你可以尝试使用&nbsp;<code>apropos</code>&nbsp;命令加上你想要干的事情的描述。比如，<code>apropos build filesystem</code>&nbsp;将会返回一系列名字和描述包括了 “build” 和 “filesystem” 单词的工具。</p><p><code>apropos</code>&nbsp;命令接受一个或多个字符串作为参数，但同时它也有其他参数，比如你可以使用&nbsp;<code>-r</code>&nbsp;参数，从而通过正则表达式来搜索。</p><h3>一个允许你来管理系统版本的替代系统</h3><p>如果你曾进行过软件开发，你就会明白跨项目管理不同版本的语言的支持的重要性。许多 Linux 发行版都有工具可以来处理不同的内建版本。</p><p>可执行文件比如&nbsp;<code>java</code>&nbsp;往往符号链接到目录&nbsp;<code>/etc/alternatives</code>&nbsp;下。反过来，该目录会将符号链接存储为二进制文件并提供一个管理这些链接的接口。Java 可能是替代系统最常管理的语言，但是，经过一些配置，它也可以作为其他应用程序替代品，比如 NVM 和 RVM （NVM 和 RVM 分别是 NodeJS 和 Ruby 的版本管理器）。</p><p>在基于 Debian 的系统中，你可以使用&nbsp;<code>update-alternatives</code>&nbsp;命令创建和管理这些链接。在 CentOS 中，这个工具就叫做&nbsp;<code>alternatives</code>&nbsp;。通过更改你的 alternatives 文件中的链接，你便可以安装一个语言的多个版本，并且在不同的情况下使用不同的二进制。这个替代系统也提供了对任何你可能在命令行运行的程序的支持。</p><h3><code>shred</code>&nbsp;命令是更加安全的删除文件方式</h3><p>我们大多数时候总是使用&nbsp;<code>rm</code>&nbsp;命令来删除文件。但是文件去哪儿了呢？真相是&nbsp;<code>rm</code>&nbsp;命令所做的事情并不是像你所想像的那样，它仅仅删除了文件系统和硬盘上的数据的硬链接。硬盘上的数据依旧存在，直到被另一个应用重写覆盖。对于非常敏感的数据来说，这会带来一个很大的安全隐患。</p><p><code>shred</code>&nbsp;命令是&nbsp;<code>rm</code>&nbsp;命令的升级版。当你使用&nbsp;<code>shred</code>&nbsp;命令删除一个文件之后，文件中的数据会被多次随机覆写。甚至有一个选项可以在随机覆写之后对所有的数据进行清零。</p><p>如果你想安全的删除一个文件并且以零覆盖，那么可以使用下面的命令：</p><pre class="brush:shell;toolbar: true; auto-links: false;">shred&nbsp;-u&nbsp;-z&nbsp;[file&nbsp;name]</pre><p>同时，你也可以使用&nbsp;<code>-n</code>&nbsp;选项和一个数字作为参数，从而指定在随机覆盖数据的时候迭代多少次。</p><h3>通过自动更正来避免输入很长的无效文件路径</h3><p>有多少次，你输入一个文件的绝对路径，然而却看到“没有该文件或目录”的消息。任何人都会明白输入一个很长的字符串的痛苦。幸运的是，有一个很简单的解决办法。</p><p>内建的&nbsp;<code>shopt</code>&nbsp;命令允许你设置不同的选项来改变 shell 的行为。设置&nbsp;<code>cdspell</code>&nbsp;选项是避免输入文件路径时一个字母出错的头痛的一个简单方式。你可以通过运行&nbsp;<code>shopt -s cdspell</code>&nbsp;命令来启用该选项。启用该选项后，当你想要切换目录时，会自动更正为最匹配的目录。</p><p>Shell 选项是节省时间的一个好方法（更不用说减少麻烦），此外还有许许多多的其他选项。如果想查看你的系统中所有选项的完整列表，可以运行不带参数的&nbsp;<code>shopt</code>&nbsp;命令。需要注意的是，这是 bash 的特性，如果你运行 zsh 或者其他可供选择的 shell，可能无法使用。</p><h3>通过子 shell 返回到当前目录</h3><p>如果你曾经配置过一个比较复杂的系统，那么你可能会发现你需要频繁的更换目录，从而很难跟踪你所在的位置。如果在运行完一个命令后自动返回到当前位置，不是很好吗？</p><p>Linux 系统实际上提供了一个解决该问题的方法，并且非常简单。如果你想通过&nbsp;<code>cd</code>&nbsp;命令进入另一个目录完成一些任务，然后再返回当前工作目录，那么你可以将命令置于括号中。你可以在你的 Linux 系统上尝试下面这个命令。记住你当前的工作目录，然后运行：<br/></p><pre class="brush:shell;toolbar: true; auto-links: false;">(cd&nbsp;/etc&nbsp;&amp;&amp;&nbsp;ls&nbsp;-a)</pre><p>该命令会输出&nbsp;<code>/etc</code>&nbsp;目录的内容。现在，检查你的当前工作目录。它和执行该命令前的目录一样，而不是&nbsp;<code>/etc</code>&nbsp;目录。</p><p>它是如何工作的呢？运行一个括号中的命令会创建一个子 shell 或一个当前 shell 进程的复刻副本。该子 shell 可以访问所有的父变量，反之则不行。所以请记住，你是在运行一个非常复杂的单行命令。</p><p>在并行处理中经常使用子 shell ，但是在命令行中，它也能为你带来同样的力量，从而使你在浏览文件系统时更加容易。</p><p>作者简介：</p><p>Phil Zona 是 Linux Academy 的技术作家。他编写了 AWS、Microsoft Azure 和 Linux 系统管理的指南和教程。他同时也管理着 Cloud Assessments 博客，该博客旨在帮助个人通过技术实现他们的事业目标。<br/></p><p>来自：&nbsp;</p>
