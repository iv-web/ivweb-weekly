<h1 id="#title_8" >9、报告显示：组件是 Java 应用易受黑客攻击的关键</h1>

[https://www.oschina.net/news/90078/state-of-software-security-report](https://www.oschina.net/news/90078/state-of-software-security-report)
<p>Java 开发者想要尽可能避免安全漏洞，应该更加了解他们在应用程序中使用的开源组件。Veracode&nbsp;公司近日发布了一份新的，数据显示 88％ 的 Java 应用包含一个以上的易受攻击的组件，还有约 53.3％的 Java 应用依赖于易受攻击的 Commons Collections 组件版本。</p><p>Veracode&nbsp;的 CTO Chris Wysopal&nbsp;表示：“组件的普遍使用，意味着当某个组件中的漏洞被披露时，可能会影响数以千计的应用”。</p><p>据该公司称，应用变得脆弱的主要原因是开发者在组件发布新版本或发现新漏洞时，多数情况下不会去修补应用。“2017年软件安全报告”也指出，只有 28％ 的团队会及时去跟踪和监测其应用使用的组件。当应用大约 75％ 的代码都是由开源组件组成时，问题就来了。</p><p>Wysopal&nbsp;表示，“开发团队不能也不应该停止使用组件，但当漏洞被利用时，时间是至关重要的。开源组件和第三方组件并不一定没有内部开发的代码安全，而是需要保持正在使用的组件及时更新“。 Wysopal&nbsp;建议公司更重视这一威胁，并使用工具来监控组件的情况。</p><p><img alt="" height="506" data-cke-saved-src="https://static.oschina.net/uploads/space/2017/1029/193159_RlNh_2896879.png" src="https://static.oschina.net/uploads/space/2017/1029/193159_RlNh_2896879.png" width="594"/></p>
